Spring Annotations — Pismo Project
====================================

====================
 APLICAÇÃO
====================

@SpringBootApplication
  Ponto de entrada da aplicação. Combina três anotações em uma:
  - @Configuration: marca a classe como fonte de beans do Spring
  - @EnableAutoConfiguration: ativa a configuração automática baseada nas dependências do classpath
  - @ComponentScan: faz o Spring escanear e registrar automaticamente os componentes do pacote atual e subpacotes
  Usado em: Main.java


====================
 CAMADA WEB (Controller)
====================

@RestController
  Combina @Controller + @ResponseBody.
  Indica que a classe é um controlador REST e que todos os métodos retornam dados diretamente no corpo da resposta HTTP (JSON), sem renderizar views.
  Usado em: ClientAccountController.java, TransactionController.java

@RequestMapping
  Define o prefixo de URL base para todos os endpoints de um controller.
  Exemplo: @RequestMapping("/accounts") faz todos os métodos daquele controller responderem a /accounts.
  Usado em: ClientAccountController.java, TransactionController.java

@GetMapping
  Atalho para @RequestMapping(method = RequestMethod.GET).
  Mapeia requisições HTTP GET para um método específico.
  Usado em: ClientAccountController.java

@PostMapping
  Atalho para @RequestMapping(method = RequestMethod.POST).
  Mapeia requisições HTTP POST para um método específico.
  Usado em: ClientAccountController.java, TransactionController.java

@PathVariable
  Extrai um valor da URL e injeta como parâmetro do método.
  Exemplo: /accounts/{id} → @PathVariable Long id captura o valor de {id}.
  Usado em: ClientAccountController.java

@RequestBody
  Desserializa o corpo da requisição HTTP (JSON) e injeta como parâmetro do método.
  O Spring usa o Jackson para converter o JSON no objeto Java correspondente.
  Usado em: ClientAccountController.java, TransactionController.java

@Valid
  Dispara a validação do Bean Validation (JSR-380) no objeto anotado.
  Sem esta anotação, as constraints (@NotNull, @Pattern, etc.) no DTO são ignoradas.
  Usado em conjunto com @RequestBody.
  Usado em: ClientAccountController.java, TransactionController.java


====================
 TRATAMENTO DE ERROS
====================

@RestControllerAdvice
  Marca a classe como um handler global de exceções para todos os controllers.
  Combina @ControllerAdvice + @ResponseBody, permitindo retornar respostas JSON nos handlers de erro.
  Usado em: GlobalExceptionHandler.java

@ExceptionHandler
  Marca um método para tratar um tipo específico de exceção lançada por qualquer controller.
  Exemplo: @ExceptionHandler(RuntimeException.class) intercepta qualquer RuntimeException.
  Usado em: GlobalExceptionHandler.java


====================
 CAMADA DE SERVIÇO
====================

@Service
  Marca a classe como um componente de serviço no contexto do Spring.
  É detectado pelo @ComponentScan e registrado como bean, permitindo injeção de dependência.
  Semanticamente indica que a classe contém lógica de negócio.
  Usado em: ClientAccountService.java, TransactionService.java


====================
 CAMADA DE PERSISTÊNCIA (Model / JPA)
====================

@Entity
  Marca a classe como uma entidade JPA, ou seja, ela será mapeada para uma tabela no banco de dados.
  Obrigatória para que o Hibernate reconheça a classe.
  Usado em: ClientAccountModel.java, OperationTypeModel.java, TransactionModel.java

@Table
  Define o nome da tabela no banco de dados que a entidade representa.
  Sem esta anotação, o Hibernate usa o nome da classe como nome da tabela.
  Exemplo: @Table(name = "accounts")
  Usado em: ClientAccountModel.java, OperationTypeModel.java, TransactionModel.java

@Id
  Marca o campo como chave primária da entidade.
  Obrigatório em toda entidade JPA.
  Usado em: ClientAccountModel.java, OperationTypeModel.java, TransactionModel.java

@GeneratedValue
  Define a estratégia de geração automática do valor da chave primária.
  strategy = GenerationType.IDENTITY delega a geração ao banco (ex: SERIAL / AUTO_INCREMENT no PostgreSQL).
  Não usado em OperationTypeModel pois os IDs são fixos e inseridos via data.sql.
  Usado em: ClientAccountModel.java, TransactionModel.java

@Column
  Configura o mapeamento de um campo para uma coluna específica no banco.
  Permite definir: nome da coluna, se é nullable, se é unique, entre outros.
  Exemplo: @Column(name = "document_number", nullable = false, unique = true)
  Usado em: ClientAccountModel.java, OperationTypeModel.java, TransactionModel.java

@PrePersist
  Marca um método para ser executado automaticamente pelo JPA antes de inserir a entidade no banco.
  Usado para popular o campo eventDate com o timestamp atual no momento da criação da transação.
  Usado em: TransactionModel.java


====================
 VALIDAÇÃO (Bean Validation)
====================

@NotNull
  Garante que o campo não seja nulo. Não valida se está vazio.
  Usado em campos de tipo objeto (Long, BigDecimal) no TransactionRequest.
  Usado em: TransactionRequest.java

@Pattern
  Valida que o campo String segue uma expressão regular específica.
  Exemplo: @Pattern(regexp = "\\d{11}") garante exatamente 11 dígitos numéricos.
  Usado em: ClientAccountRequest.java
